# ヒアリングなし提案パターン

## 🎯 3つのアプローチ

### 1. **エクスプレス提案** (最速1日)

```bash
# URL分析 → 即提案書生成
npm run skilldex analyze -u https://example.com
npm run skilldex generate-proposal
```

**特徴**:
- Web情報のみで提案
- 汎用的なスキルを推薦
- 概算見積もり
- すぐに商談可能

**適している場合**:
- 初回アプローチ
- 競合コンペ
- 予算確認段階

---

### 2. **リサーチ強化提案** (3-5日)

```bash
# 複数ソースから情報収集
npm run skilldex hybrid \
  -u https://company.com \
  -u https://company.com/news \
  -u https://company.com/ir \
  -f industry-report.md
```

**追加調査項目**:
- 業界レポート
- 競合他社分析
- ニュースリリース
- 決算情報（上場企業）

**特徴**:
- より精度の高い提案
- 業界特有の課題に対応
- 競合優位性の明示

---

### 3. **仮説検証型提案** (7-10日)

```bash
# 仮説を立てて提案
npm run skilldex analyze \
  --hypothesis "残業削減が最優先課題" \
  --target-roi "6months"
```

**プロセス**:
1. Web分析から仮説構築
2. 仮説に基づくスキル選定
3. シナリオ別の効果試算
4. A/Bパターンの提案

**特徴**:
- 複数シナリオ提示
- リスク分析込み
- 段階的検証プラン

## 📊 提案精度と成約率の関係

| アプローチ | 準備期間 | 提案精度 | 成約率 | コスト |
|-----------|---------|----------|--------|--------|
| エクスプレス | 1日 | 60% | 20% | 低 |
| リサーチ強化 | 3-5日 | 75% | 35% | 中 |
| 仮説検証型 | 7-10日 | 85% | 50% | 高 |
| ヒアリング込み | 14日+ | 95% | 70% | 最高 |

## 🚀 使い分けガイドライン

### エクスプレス提案を選ぶべき時
- 大量の見込み客にアプローチ
- まず興味を引きたい
- 予算感を探りたい
- スピード重視の案件

### リサーチ強化提案を選ぶべき時
- ある程度本気度の高い見込み客
- 業界特性が重要な場合
- 競合との差別化が必要
- 中規模案件（500-2000万円）

### 仮説検証型提案を選ぶべき時
- 大型案件（2000万円以上）
- 経営層へのアプローチ
- 複数部門が関わる
- 長期的な関係構築

## 💡 成功のコツ

1. **期待値調整**
   - 「初期提案」であることを明示
   - ヒアリングの重要性も伝える
   - 柔軟に修正可能と強調

2. **フォローアップ**
   - 提案後1週間以内に連絡
   - 簡易ヒアリングの提案
   - 追加情報の提供

3. **データ蓄積**
   - 提案の反応を記録
   - 成功パターンの分析
   - スキル精度の向上
